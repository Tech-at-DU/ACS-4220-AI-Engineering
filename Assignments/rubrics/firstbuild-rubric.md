# Firstbuild Rubric

Source: [../firstbuild.md](../firstbuild.md)

## Paste-Ready Markdown

```md
## Project Setup

- **Needs Improvement**: Missing key files or incomplete git history
- **Basic**: `CLAUDE.md`, `spec.md`, and basic repo structure present
- **Proficient**: Clear, well-organized project with 5+ acceptance criteria and 3+ quality gates
- **Advanced**: Setup serves as a template for future projects

## Test-First Workflow

- **Needs Improvement**: Tests written after code, or tests missing
- **Basic**: Tests present but incomplete; some features lack test coverage
- **Proficient**: Full red/green/refactor cycle visible in commit history for 3+ features
- **Advanced**: Sophisticated test patterns; advanced refactoring strategies evident

## Feature Completion

- **Needs Improvement**: Incomplete features; failing tests
- **Basic**: Core features working; most tests passing
- **Proficient**: All features complete, all tests passing, code is clean
- **Advanced**: Thoughtful complexity handling; multiple refactoring cycles

## Commit Quality

- **Needs Improvement**: Poor organization or missing commits
- **Basic**: Adequate commits; some squashing needed
- **Proficient**: Clear commit messages showing progression; tests before implementation
- **Advanced**: Excellent narrative arc in commit history; evidence of refinement

```
