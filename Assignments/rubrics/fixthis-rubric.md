# Fixthis Rubric

Source: [../fixthis.md](../fixthis.md)

## Paste-Ready Markdown

```md
## Codebase Understanding

- **Needs Improvement**: Minimal exploration; architecture unclear
- **Basic**: Basic mapping of codebase; some patterns identified
- **Proficient**: Thorough analysis of architecture, patterns, and conventions
- **Advanced**: Deep understanding reflected in clear, well-scoped rules

## Configuration Quality

- **Needs Improvement**: Missing `CLAUDE.md` or `AGENTS.md`
- **Basic**: Present but generic instructions
- **Proficient**: Comprehensive, project-specific guidance with clear conventions
- **Advanced**: Config serves as a template; other students could clone and build immediately

## Rules & Enforcement

- **Needs Improvement**: No rules or rules don't match project patterns
- **Basic**: Basic rules written; limited testing
- **Proficient**: 3+ well-scoped rules enforcing existing conventions; tested
- **Advanced**: Advanced rules combining testing patterns and architectural constraints

## Feature Implementation

- **Needs Improvement**: Feature incomplete or breaks existing tests
- **Basic**: Feature works; some test failures
- **Proficient**: All tests pass; feature follows project patterns; no regressions
- **Advanced**: Sophisticated implementation; demonstrates mastery of project patterns

## Documentation

- **Needs Improvement**: Minimal or unclear writeup
- **Basic**: Basic exploration notes in `writeup.md`
- **Proficient**: Clear documentation of exploration, tuning, and results
- **Advanced**: Excellent before/after evidence of context refinement

```
