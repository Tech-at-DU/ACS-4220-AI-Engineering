# Wiredup Rubric

Source: [../wiredup.md](../wiredup.md)

## Paste-Ready Markdown

```md
## Protocol Configuration

- **Needs Improvement**: Incomplete or missing server setup
- **Basic**: 2 servers configured; basic functionality
- **Proficient**: 2+ servers properly configured and integrated; clear `.mcp.json`
- **Advanced**: 3+ servers; custom tools built; advanced configuration

## Live Documentation

- **Needs Improvement**: No evidence of live docs usage
- **Basic**: Docs used occasionally; limited impact captured
- **Proficient**: Evidence of live doc queries reducing hallucinations; clear before/after in writeup
- **Advanced**: Quantified improvement; demonstrates mastery of live context

## Integration Quality

- **Needs Improvement**: Minimal real-world use
- **Basic**: Servers used in 1-2 tasks; some friction evident
- **Proficient**: Servers integrated seamlessly into workflow; clear impact on development
- **Advanced**: Sophisticated integration; evidence of workflow optimization

## Evidence & Documentation

- **Needs Improvement**: Missing or unclear evidence
- **Basic**: Basic evidence in files; minimal comparison
- **Proficient**: Clear, well-organized evidence; side-by-side hallucination comparison
- **Advanced**: Compelling narrative showing tangible impact on code quality

## Feature Implementation

- **Needs Improvement**: Incomplete or broken tests
- **Basic**: Feature works; some test issues
- **Proficient**: All tests pass; feature well-designed; uses protocol servers meaningfully
- **Advanced**: Sophisticated design; advanced use of multiple integrations

```
