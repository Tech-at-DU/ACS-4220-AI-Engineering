# makeanything Project Rubric

Source: [../makeanything.md](../makeanything.md)

## Paste-Ready Markdown

```md
## Scope & Ambition

- **Needs Improvement**: Trivial project or incomplete implementation
- **Basic**: Meaningful application demonstrating 3-4 course skills
- **Proficient**: Meaningful application demonstrating 5+ course skills
- **Advanced**: Ambitious project with creative application of advanced techniques

## Spec Quality

- **Needs Improvement**: Missing or vague descriptions
- **Basic**: Basic descriptions present
- **Proficient**: Quality gates with structured, verifiable acceptance criteria
- **Advanced**: Specs thorough enough that another student could reproduce the build

## Workflow Mastery

- **Needs Improvement**: Ad-hoc development without consistent process
- **Basic**: Some test-first work, inconsistent approach
- **Proficient**: Full test-first workflow with appropriate complexity awareness
- **Advanced**: Sophisticated use of design exploration, specialized workers, and parallel execution

## Tool Configuration

- **Needs Improvement**: Minimal or generic setup
- **Basic**: Basic configuration present
- **Proficient**: Rich, project-specific configuration with evidence of refinement
- **Advanced**: Config serves as a reusable template for similar projects

## Presentation

- **Needs Improvement**: Unprepared or unable to explain workflow decisions
- **Basic**: Demo functional; basic walkthrough
- **Proficient**: Clear demo, workflow walkthrough, and lessons learned
- **Advanced**: Engaging presentation with insights that benefit the whole class

## Work Ethic

- **Needs Improvement**: < 10 commits, work done last minute
- **Basic**: Sporadic commits
- **Proficient**: Consistent commits across the project period
- **Advanced**: Iterative refinement visible in commit history

```
