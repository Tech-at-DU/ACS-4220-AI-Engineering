## How to Read This

This changelog is written for students. Each entry focuses on what changed in the course, why it matters, and what you should expect when you open the syllabus, lesson plans, assignments, and supporting materials.

This version is based on git commit history plus the current working-tree changes. The GitHub repository currently has no pull request history available to incorporate.

The format follows a simple pattern:

1. `Added` for brand-new materials or features.
1. `Changed` for revised structure, wording, deadlines, or expectations.
1. `Fixed` for corrections, cleanup, or consistency improvements.

## Unreleased

Changes prepared in the current working tree after the most recent commit.

**Student Impact**

The course materials are easier to navigate, assignment expectations are clearer, and the lesson plans are more consistently structured across the semester.

**Added**

- Lesson agendas at the top of every lesson plan so you can scan the structure before class starts.
- Separate rubric export files for instructor workflows while keeping the student-facing rubric tables inside each assignment.
- An instructor guide for course operations and Gradescope planning.

**Changed**

- Assignment intros now use the `Why Should I Do This?` framing instead of a generic assignment-description heading.
- Gradescope references now use the linked word [Gradescope](https://www.gradescope.com/courses/1293005) consistently across the syllabus and assignment submissions.
- The syllabus now includes an instructor automation section and clearer navigation to supporting documentation.
- The course now includes a Node-based planning scaffold for Gradescope setup, including structured rubric JSON and a dry-run planner.

**Fixed**

- Markdown linting is now configured at the repo level, with a repeatable command for checking Markdown quality.
- Lesson plans now use a consistent agenda pattern and cleaner challenge anchors.
- Duplicate project instructions in `AGENTS.md` and `.claude/CLAUDE.md` were reduced so the repo rules are easier to maintain.
- CSV rubric exports were removed because the current Gradescope workflow does not use them.
